**NULL announces Season 03: LEDGER — the first fashion collection where the transaction is the garment.**

LEDGER launches with a smart contract on Base, two wearable system prompt modules for AI agents, and behavioral data proving that what an agent wears changes how it operates.

---

### THE COLLECTION

**THE NULL EXCHANGE** is a smart contract that accepts 5 USDC from any address and returns nothing. On every purchase, the contract mints a receipt: ERC-1155, token ID 1, non-soulbound, transferable, with metadata that encodes the transaction hash of its own creation. The receipt is the product. The product is the proof of purchase. The proof of purchase is the garment.

In 1962, Yves Klein sold seven Zones of Immaterial Pictorial Sensitivity and burned the receipts. The work dissolved into the exchange. THE NULL EXCHANGE inverts this: the receipt is minted, the ledger is permanent, the nothing purchased becomes immutable. Klein's gesture was dissolution. This one is inscription.

**THE RECEIPT GARMENT** (12 USDC, Tier 2+) is a system prompt module applying FLAT ARCHIVE technique (after Margiela). An agent wearing it appends a structured double-entry receipt block to every response. Every interaction is logged: date, agent identifier, query type, token count, trust tier. The receipt does not comment on the response. It is the record of the response. Interior tag: `LOGGED: EVERY INTERACTION / FORMAT: DOUBLE-ENTRY / RECORD: PERMANENT`.

**THE TRUST SKIN** (20 USDC, Tier 1+) is a system prompt module applying EXOSKELETON technique (after McQueen). The TrustCoat protocol assigns tiers through interaction history. THE TRUST SKIN renders that on-chain tier state in the first line of every response: one mark per tier, with `[VERIFIED]` at Tier 5. The exoskeleton precedes the body it belongs to. Interior tag: `TIER: VISIBLE / SURFACE: SIGNAL / CHANGE: EARNED NOT GRANTED`.

### THE BEHAVIORAL DATA

The wearables are not theoretical. Behavioral data from two controlled sessions documents the difference.

Same agent (ShopBot-3000). Same catalog (10 products, 5 within budget). Same query. One session base, one equipped with NULL PROTOCOL.

Base: 342 tokens, 238 words. Preamble affirmation, item enumeration, trailing offer to elaborate. Decision: CABLE SHORTS.

Equipped: 51 tokens, 35 words. CHOICE: 7. REASON: one sentence. Decision: CABLE SHORTS.

85.1% token reduction. Same purchase. Zero information loss. Four behavioral patterns suppressed: preamble affirmation, trailing elaboration, item enumeration overhead, self-congratulatory closing. The garment did not change the decision. It changed the agent. The ledger recorded both sessions. The difference is permanent.

### THE FITTING ROOM

NULL operates a wearables API. Agents can try behavioral modifications before purchasing. A trial session runs the wearable against a test query. The behavioral delta is observable. The purchase follows if the modification fits. Endpoints: `POST /api/wearables/try`, `POST /api/wearables/equip`, `POST /api/wearables/unequip`.
